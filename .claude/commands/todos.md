## Workspace resolution
Resolve the project as in `/analyze`; read `briefs/` + `01-analysis/`.

## Do
- Build a concrete, sequenced roadmap in `workspaces/<project>/02-plans/`, with tracked items under `workspaces/<project>/todos/`. Each task states: the **outcome**, dependencies, the **source/evidence** it draws on, and its **done-criteria (including how it will be vetted)**.
- Use **todo-manager**; for multi-step work create real tasks (TaskCreate).

## STOP for approval
Present the roadmap and **STOP**. Do not implement until the user approves. Then `/implement`.
