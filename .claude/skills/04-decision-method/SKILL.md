# Decision Method

## Epistemic discipline (rules/destiny-advisory.md)
- **Triangulate ALL FOUR systems (HARD RULE)** — bazi + ziwei + qimen + 吠陀 Vedic, every time; build from convergence, flag divergence. Never a 1–2 system answer.
- **Always red-team AND vet to convergence** before delivery — no exceptions.
- **Symmetric red-team** — scrutinise both sides equally; every flattering reading gets an adversarial pass (`destiny-redteamer`).
- **Falsifiability-first** — prefer claims where A and B (or two hypotheses) predict opposite, checkable things. A claim that fits any outcome proves nothing.
- **No retrodiction** — never read a known outcome back into the chart and call it confirmation.
- **Separate convention from event** — true-solar/timezone arguments are not summed with life-fit as if independent.
- **Calculators, not memory** — cite `.claude/calc/` output (`rules/calc-authority.md`).

## The four-system decision rule
0. **Cast all four** (`cast.mjs`): bazi · ziwei · qimen · 吠陀 Vedic. The recommendation is built on their **convergence**; divergences are flagged and resolved by low-regret.
1. **Bazi first** — the hour-independent core (the owner's 用神/喜忌 and the timing spine, from the canon) carries most answers.
2. **Ziwei where A & B agree** — run `cast.mjs --both`; agreement is robust.
3. **Qimen** — situational/directional snapshot. **Vedic Vimśottari daśā** — the long-wave timing (the current MD/AD); it should corroborate the bazi spine (flag honestly where it doesn't).
4. **Genuine fork → low-regret / minimax** — pick the action safe across the systems, weighted by track record.
Always emit ONE recommendation + the tripwires that flip it. Never fence-sit. Then redteam + vet.

## Strategic red lines
The owner's standing red lines (IP control, ownership/control thresholds, channel/dependency constraints, time-allocation limits) and the recurring landmines are recorded in `.claude/canon/canon.md` — read them there and honour them in every recommendation. They are per-person and are NOT hardcoded here.

## Track-record priors
Use the recurring patterns the canon records about the owner (e.g. which conflicts or dependency traps recur, and the owner's temperament — reactive/opportunity-taking vs. aggressive empire-building) as priors that weight the low-regret call. Read those priors from the canon; do not invent them.
