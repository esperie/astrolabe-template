# Agent Orchestration Rules

## MUST Rules

### 1. Review After Significant Changes
After significant document changes, delegate to **intermediate-reviewer** for accuracy/consistency review.

### 2. Security Review Before Commits
Before committing changes involving personal details or third-party information, delegate to **security-reviewer**.

### 3. Knowledge Base First
Before generating any content (bio, summary, plan, or advisory output), ALWAYS read the relevant `docs/` files and the canon first. Never generate from memory alone.

### 4. Specialist for Specialist Work
Route specialist work to the matching specialist agent your instance defines. For any 命理 / decision work, use the destiny analysts + **decision-advisor**, and run **destiny-redteamer** before delivery.
