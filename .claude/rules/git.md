# Git Workflow Rules

## MUST Rules

### 1. Conventional Commits
Format: `type(scope): description`
Types: feat, fix, docs, style, refactor, chore
Scopes: knowledge-base, cv, branding, media, agents, skills, rules

### 2. No Secrets in Commits
Never commit files containing passwords, API keys, or sensitive personal information beyond what's in the knowledge base.

### 3. Private Repository
MUST NOT change repository visibility to public.
