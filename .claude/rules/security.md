# Security Rules

## MUST Rules

### 1. Private Repository
This repository MUST remain private on GitHub. It contains personal details, client names, and strategic information.

### 2. No Secrets in Content
Generated CVs, bios, and public content MUST NOT contain:
- Personal phone numbers (unless explicitly requested)
- Home addresses
- Financial details (revenue, salary, investment specifics)
- Client contract details
- Strategic plans or competitive intelligence

### 3. Client Confidentiality
Client names may only be used where:
- Public evidence exists linking the client to the instance owner or their company
- Or explicit permission has been granted
- Approved client names are listed in `docs/04-clients/README.md`
